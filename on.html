    Using Moloch to Write Better Network Detections
    Signature based network detections are only as good as the signatures used to inspect network traffic. Analysts commonly will write network signatures for IDS/IPS engines such as Suricata, based off of a single malware sample or PCAP. ...more  This, unfortunately, can lead to false positivesor arguably worse: false negatives during traffic analysis. By leveraging Moloch to cluster PCAPs from multiple, sometimes hundreds of unique sandbox runs for the same family of malware, it allows us to obtain a better understanding of what is or isn't unique in traffic. Based off this we can write better stronger detections that can outlast basic variance in traffic patterns, updates in malware C2 protocol, and spot gaps in detection. less 
    Jack Mott & Jason Williams

    More Data is Good, Good Data is Better
    What do albums, 8-track tapes, cassettes, CDs and mp3s have in common with network operational data? Network operational data has gone through the same “transformation”, ...more  namely, it’s the same data being delivered via the next generational transport systems (eg syslog, show commands, SNMP, Openconfig, streaming telemetry). While existing network data is adept at reporting binary conditions, (eg link up/down, bgp up/down, etc.), this existing data appears to be inadequate for determining more subtle network degradations or failures. Using real-life undetected network events as use cases, this talk suggests there’s an untapped high fidelity stream of network operational data that service providers are collectively overlooking, namely the processing of router control plane traffic. This talk shows how the router control plane traffic is a haystack of needles and how we are augmenting Moloch to make it network domain aware. less 
    Brian Field
